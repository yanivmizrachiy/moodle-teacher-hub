# Moodle Teacher Hub

מוצר מורה בעברית RTL עבור נתוני Moodle אמיתיים.

## מצב אסטרטגי עדכני

נכון לעכשיו אין Moodle Web Services API token / Security Key / API key.

לכן המצב הפעיל של המוצר הוא:

**Manual Real-Data Mode Without API Token**

המערכת אינה מושכת נתונים אוטומטית מ־Moodle, אינה מבקשת שם משתמש/סיסמה של Moodle, ואינה מציגה נתוני דמו.

הנתונים שמוצגים בממשק חייבים להגיע רק מנתוני Moodle אמיתיים שהמורה סיפק ידנית:

- קובץ Excel/XLSX שהורד מ־Moodle
- CSV
- ODS אם הדפדפן מצליח לקרוא אותו
- TXT/TSV
- טבלה שהועתקה מתוך דוח Moodle והודבקה לממשק

## מה כבר קיים בפועל

- שרת Express בסיסי.
- מסלול LTI 1.1 לפתיחת הכלי מתוך Moodle.
- לכידת הקשר כניסה מ־LTI, כאשר Moodle מספק אותו.
- דשבורד עברי RTL.
- אשף ייבוא בשם: `ייבוא נתוני Moodle אמיתיים`.
- ייבוא מקומי בצד הדפדפן דרך `ManualImportAdapter`.
- שמירה מקומית בדפדפן ב־localStorage.
- תצוגה מקדימה לפני ייבוא.
- מיפוי עמודות ידני לאחר זיהוי ראשוני.
- מסכי תלמידים, משימות, פרקים, ציונים, פעילות/זמן, דוחות, ייצוא ומקורות נתונים — על בסיס הנתונים שיובאו.
- ייצוא CSV לציונים ולפעילות.
- הדפסה / שמירה כ־PDF דרך הדפדפן.

## סוגי דוחות Moodle שהממשק מיועד לקבל

- Gradebook / Grade export
- Logs / יומנים
- Activity Completion
- Activity Report
- Participation Report
- Quiz attempts / grades
- Assignment submissions / grades
- Course page / activity list שהועתק מ־Moodle

## כללי אמת מחייבים

- אין נתוני דמו.
- אין תלמידים מזויפים.
- אין משימות מזויפות.
- אין ציונים מזויפים.
- אין פעילות או זמן מזויפים.
- חוסר ציון נשאר חוסר ציון, לא אפס.
- מספר שאלות מוצג רק אם קיים בדוח שיובא.
- משך עבודה מדויק מוצג רק אם קיים שדה משך רשמי. אם יש רק חותמות זמן, מוצג חלון פעילות מחושב ולא משך רשמי.

כאשר נתון חסר, הממשק צריך להציג ניסוחים כמו:

- `חסר מהנתונים שיובאו`
- `לא זמין בקובץ הזה`
- `נדרש דוח נוסף ממודל`
- `לא ניתן לחשב ללא לוגים`

## מה עדיין לא קיים / לא מאומת

- אין כרגע סנכרון Moodle API חי.
- אין משיכת ציונים/יומנים אוטומטית מ־Moodle.
- אין MoodleWsAdapter פעיל כי אין טוקן.
- אין מסד נתונים שרת קבוע לייבוא הידני; הייבוא הנוכחי נשמר מקומית בדפדפן.
- אין אימות production מלא ל־LTI מעבר למה שקיים בשרת הנוכחי.
- אין בדיקות אוטומטיות מקיפות לקבצי Moodle גדולים.

## כיוון המשך

הארכיטקטורה צריכה להישאר תואמת לשני adapters:

1. `ManualImportAdapter` — המצב הפעיל כעת.
2. `MoodleWsAdapter` — עתידי בלבד, אם יתקבל טוקן Moodle Web Services אמיתי.

שני ה־adapters צריכים להזין בעתיד אותו מודל נתונים מנורמל ואותם מסכי UI, בלי לפצל לוגיקה בין מצב ידני למצב API.

## הפעלה מקומית

```bash
npm install
npm run dev
```

לאחר הפעלה מקומית:

```text
http://127.0.0.1:3000/dev/login
```

או דרך LTI 1.1 כאשר מוגדרים ערכי הסביבה המתאימים.
